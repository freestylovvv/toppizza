# 🗄️ Создание базы данных PostgreSQL

## Способ 1: Через командную строку (psql)

### Шаг 1: Открыть psql
```bash
psql -U postgres
```
Введите пароль от пользователя postgres

### Шаг 2: Создать базу данных
```sql
CREATE DATABASE toppizza;
```

### Шаг 3: Проверить создание
```sql
\l
```
Вы должны увидеть `toppizza` в списке баз данных

### Шаг 4: Выйти из psql
```sql
\q
```

---

## Способ 2: Через pgAdmin (графический интерфейс)

### Шаг 1: Открыть pgAdmin
Запустите pgAdmin 4 из меню Пуск

### Шаг 2: Подключиться к серверу
- Раскройте "Servers" в левой панели
- Введите пароль если требуется

### Шаг 3: Создать базу данных
1. Правый клик на "Databases"
2. Выберите "Create" → "Database..."
3. В поле "Database" введите: `toppizza`
4. Нажмите "Save"

---

## Способ 3: Через командную строку Windows (createdb)

```bash
createdb -U postgres toppizza
```
Введите пароль когда попросит

---

## Проверка подключения

После создания базы данных проверьте файл `.env`:

```env
DATABASE_URL="postgresql://postgres:ВАШ_ПАРОЛЬ@localhost:5432/toppizza?schema=public"
```

Замените:
- `postgres` - на ваше имя пользователя PostgreSQL
- `ВАШ_ПАРОЛЬ` - на ваш пароль
- `localhost` - если PostgreSQL на вашем компьютере
- `5432` - стандартный порт PostgreSQL

## Полная последовательность запуска проекта

### 1. Установить зависимости
```bash
npm install
```

### 2. Создать базу данных (один из способов выше)
```bash
createdb -U postgres toppizza
```

### 3. Настроить .env файл
Откройте `.env` и укажите правильные данные подключения

### 4. Сгенерировать Prisma Client
```bash
npm run prisma:generate
```

### 5. Создать таблицы в базе данных
```bash
npm run prisma:push
```

### 6. Заполнить базу данных начальными данными
```bash
npm run prisma:seed
```

### 7. Запустить проект
```bash
npm run dev
```

### 8. Открыть в браузере
http://localhost:3000

## Возможные проблемы

### Ошибка: "database does not exist"
**Решение:** База данных не создана. Создайте её одним из способов выше.

### Ошибка: "password authentication failed"
**Решение:** Неверный пароль в `.env` файле. Проверьте пароль PostgreSQL.

### Ошибка: "could not connect to server"
**Решение:** PostgreSQL не запущен. Запустите службу PostgreSQL:
- Windows: Службы → PostgreSQL → Запустить
- Или через pgAdmin

### Ошибка: "role does not exist"
**Решение:** Пользователь не существует. Используйте `postgres` или создайте нового пользователя.

## Полезные команды psql

```sql
-- Список всех баз данных
\l

-- Подключиться к базе данных
\c toppizza

-- Список всех таблиц
\dt

-- Описание таблицы
\d Product

-- Посмотреть данные
SELECT * FROM "Product";

-- Удалить базу данных (ОСТОРОЖНО!)
DROP DATABASE toppizza;
```

## Альтернатива: Использовать существующую базу

Если у вас уже есть база данных PostgreSQL, просто измените название в `.env`:

```env
DATABASE_URL="postgresql://postgres:password@localhost:5432/ВАША_БАЗА?schema=public"
```

Prisma создаст таблицы в этой базе данных.
